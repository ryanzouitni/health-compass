Goal

Improve Health Compass by fixing logic bugs, privacy issues, UX validation issues, and making outputs safer/clearer (orientation, not diagnosis). Implement changes in this repo (TypeScript/Express + React).

A) Critical privacy/security fixes (DO FIRST)
A1) Remove all logging of health data (PHI)

Problem: server/routes.ts logs full request body; server/index.ts may log JSON response bodies. This violates “we don’t store data” because logs count as storage.

Tasks

In server/routes.ts, delete or redact:

console.log("Received assessment data:", JSON.stringify(req.body, null, 2));


Replace with minimal metadata logging only, e.g.:

console.info("Assessment request received", { hasBody: !!req.body });


In server/index.ts, find any request/response logger that prints bodies, and remove body logging. Keep only:

method, path, status, duration

B) Fix the biggest scoring bugs (logic correctness)
B1) Pediatric double-counting: family history

Problem: In pediatrics logic, familyHistoryDiabetes gets +2, and later adult logic adds +3 again (same factor counted twice; also duplicated factor cards).

Task

In server/routes.ts:

When ageGroup is "infant" or "child", do not run adult-family-history scoring.

Implement a guard variable:

const isPediatric = data.ageGroup === "infant" || data.ageGroup === "child";


Then wrap adult family history scoring in:

if (!isPediatric && data.familyHistoryDiabetes) { ... }


Also ensure you do not push the same factor twice (avoid two different ids for the same thing).

B2) Pediatric symptom double-counting (thirst/urination/weight loss)

Problem: pediatric branch adds points for those symptoms, and later diabetesSymptomCount adds more points again.

Task

If isPediatric:

Either:

skip the diabetesSymptomCount step entirely, OR

ensure symptoms are scored only once by building a single symptom scoring function for pediatrics.

Implement simplest safe fix:

if (!isPediatric) {
  // existing diabetesSymptomCount scoring
}

B3) Fix gestational diabetes translation keys

Problem: gestational factor uses wrong nameKey/explanationKey (currently points to familyDiabetes keys).

Task

In server/routes.ts, replace:

id: "gestational",
nameKey: "factor.familyDiabetes",
explanationKey: "factor.familyDiabetes.explanation",


With new keys:

id: "gestationalDiabetes",
nameKey: "factor.gestationalDiabetes",
explanationKey: "factor.gestationalDiabetes.explanation",


Add these keys to the translation file(s) used by the frontend (where other factor.* keys live).

B4) Ensure every score change has a visible “why” factor

Problem: additionalSymptomCount >= 3 adds points without adding a factor card.

Task

Whenever you add points, also push a factor entry.
Example:

if (additionalSymptomCount >= 3) {
  urgencyScore += 2;
  contributingFactors.push({
    id: "multipleAdditionalSymptoms",
    nameKey: "factor.multipleAdditionalSymptoms",
    explanationKey: "factor.multipleAdditionalSymptoms.explanation",
    score: 2,
    condition: "3+ additional symptoms selected",
  });
}


Add translation keys.
